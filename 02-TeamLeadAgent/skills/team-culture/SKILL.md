# 团队文化 — team-culture

## 面试评估框架

### 评分表

| 维度 | 权重 | 评分 (1-5) |
|---|---|---|
| 技术能力 | 40% | |
| 系统设计 | 20% | |
| 沟通协作 | 15% | |
| 问题解决 | 15% | |
| 文化匹配 | 10% | |
| **总分** | 100% | ≥ 3.5 通过 |

### 行为面试题

| 维度 | 问题 |
|---|---|
| 协作 | "讲一个你和不同意见同事合作的经历" |
| 抗压 | "讲一个你在紧迫期限下完成项目的经历" |
| 成长 | "讲一个你从失败中学习的经历" |
| 主动 | "讲一个你主动发现并解决问题的经历" |

## 知识管理体系

```
团队知识库
├── 技术规范 — 编码规范、架构规范
├── 设计文档 — 架构设计、模块设计
├── 最佳实践 — 编码技巧、排错指南
├── 新人指南 — 入职流程、环境搭建
├── 事故记录 — 线上事故、分析、预防
└── 会议记录 — 技术决策、复盘
```

## 团队文化维度

| 维度 | 好的表现 | 问题表现 |
|---|---|---|
| 信任 | 敢于表达不同意见 | 沉默、背后议论 |
| 冲突 | 建设性辩论 | 人身攻击或回避 |
| 承诺 | 主动承担责任 | 推诿、等指令 |
| 责任 | 主动跟进、互相补位 | 各管各的 |
| 结果 | 关注团队目标 | 关注个人表现 |
