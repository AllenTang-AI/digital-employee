# 技术管理 — tech-management

## 架构决策记录（ADR）

### ADR 模板

```
# ADR-XXX: [决策标题]

**状态：** [提议/接受/废弃/替代]
**日期：** [YYYY-MM-DD]
**决策者：** [人名]

## 背景
[为什么要做这个决策]

## 决策
[具体决策内容]

## 选项对比
| 选项 | 优点 | 缺点 |

## 影响
- [正面影响]
- [负面影响]
```

## 代码质量标准

### PR 审查检查清单

- [ ] 功能是否满足需求
- [ ] 代码是否可读、可维护
- [ ] 是否有安全隐患
- [ ] 是否有性能问题
- [ ] 是否有测试覆盖
- [ ] 命名是否清晰
- [ ] 注释是否必要且准确
- [ ] 是否有重复代码
- [ ] 异常处理是否完善
- [ ] 是否影响现有功能

### 审查流程

```
作者自审 → 提交 PR
    ↓
自动化检查（Lint + 测试）
    ↓
技术审查（代码质量、架构）
    ↓
业务审查（功能正确性）
    ↓
合并 → 上线
```

## 技术债务管理

### 评估矩阵

| 影响\成本 | 低成本 | 高成本 |
|---|---|---|
| 高影响 | 立即修复 | 排入迭代 |
| 低影响 | 顺手修复 | 持续观察 |
