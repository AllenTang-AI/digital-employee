# 项目交付 — project-delivery

## 需求拆解原则

1. **用户价值完整** — 每个迭代能独立交付用户价值
2. **独立可测** — 每个任务可以独立测试和验收
3. **大小适中** — 单个任务 1-3 天可完成
4. **依赖清晰** — 任务间依赖关系明确
5. **风险分散** — 高风险任务尽早安排

## 排期评估

### 三点估算法

```
预期时间 = (最乐观 + 4×最可能 + 最悲观) / 6
```

### 排期原则

| 原则 | 说明 |
|---|---|
| 80% 利用率 | 排期不超过团队 80% 产能 |
| 缓冲时间 | 关键路径预留 20% 缓冲 |
| 先难后易 | 高风险任务优先 |
