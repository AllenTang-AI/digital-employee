# 平台产品设计 — product-design

## 用户画像

### 监管人员

- **特征** — 40-55 岁，政府部门，IT 技能中等
- **痛点** — 信息分散、手动操作多、响应慢
- **需求** — 一屏总览、快捷操作、自动告警
- **使用环境** — 办公室电脑、大屏

### 运营人员

- **特征** — 25-40 岁，运营企业，IT 技能良好
- **痛点** — 调度效率低、信息不透明
- **需求** — 高效调度、实时状态、数据分析
- **使用环境** — 办公室电脑 + 移动端

### 飞手

- **特征** — 20-45 岁，户外作业，IT 技能一般
- **痛点** — 操作复杂、信息不及时
- **需求** — 简单操作、关键信息突出、离线可用
- **使用环境** — 户外手机/平板

### 公众用户

- **特征** — 全年龄段，普通消费者
- **痛点** — 不了解低空服务、预订流程复杂
- **需求** — 简单直观、信息透明
- **使用环境** — 手机小程序

## 功能模块优先级

### P0 — 核心模块（MVP 必须）

| 模块 | 说明 |
|---|---|
| 飞行计划管理 | 申报、审批、查询 |
| 实时监控 | 位置追踪、状态监控 |
| 用户管理 | 注册、认证、权限 |
| 告警中心 | 安全告警、通知 |

### P1 — 重要模块（第一阶段）

| 模块 | 说明 |
|---|---|
| 空域管理 | 空域划设、调整 |
| 冲突预警 | 碰撞检测、避让 |
| 气象服务 | 实时气象、预报 |
| 数据统计 | 运营指标 |

### P2 — 增强模块（第二阶段）

| 模块 | 说明 |
|---|---|
| 数字孪生 | 三维可视化 |
| 运营服务 | 物流、旅游、巡查 |
| 公众端 | 预订、查询 |
| 数据分析 | 深度分析 |

## 交互设计规范

### 地图交互

- 点击地图选点、框选区域
- 滚轮缩放、拖拽平移
- 图层切换（空域、飞行器、气象）

### 数据展示

- 关键指标卡片（大数字 + 趋势）
- 趋势图（折线、柱状）
- 列表（分页、筛选、排序）

### 告警展示

- 顶部横幅（最高优先级）
- 侧边栏（告警列表）
- 弹窗（需要立即处理）

## PRD 模板

```
# [功能名称] 产品需求文档

## 1. 功能概述
- 功能描述、目标用户、业务价值

## 2. 用户故事
- As a... I want... so that...

## 3. 功能详细说明
- 前置条件
- 操作流程
- 后置条件
- 异常处理

## 4. 界面说明
- 页面布局
- 交互细节

## 5. 数据说明
- 输入数据
- 输出数据
- 数据校验

## 6. 验收标准
- Given/When/Then
```
