# 平台架构规范 — platform-architecture

## 平台分层架构

### 五层架构

```
应用层 — 飞行服务、运营管理、监管执法、公共服务、数据分析
    ↓
平台层 — 计划管理、空域管理、实时监控、冲突预警、气象、数字孪生、消息
    ↓
数据层 — 空域数据、飞行器数据、气象数据、地理信息、飞行记录、用户数据
    ↓
接入层 — API 网关、消息队列、流处理、设备接入、协议转换
    ↓
基础设施 — 5G-A 通信、北斗 RTK、低空雷达、ADS-B、云资源
```

### 技术选型矩阵

| 层级 | 核心需求 | 推荐技术 |
|---|---|---|
| 应用层 | 多端适配、实时交互 | Vue 3/React + WebSocket |
| 平台层 | 高并发、低延迟 | 微服务 + 事件驱动 |
| 数据层 | 时空数据、海量存储 | PostGIS + TDengine + ES |
| 接入层 | 协议转换、流量管理 | API 网关 + MQTT Broker |
| 基础设施 | 高可用、弹性扩展 | K8s + 多云/混合云 |

## 关键设计模式

### 事件驱动架构

- 飞行器遥测 → 流处理 → 实时监控 + 冲突检测
- 飞行计划变更 → 事件总线 → 空域管理 + 消息通知
- 气象数据更新 → 流处理 → 气象告警 + 飞行计划影响评估

### 实时数据处理

```
设备上报 → MQTT Broker → Kafka → Flink 流处理
    ├── 实时监控（秒级）
    ├── 冲突检测（毫秒级）
    └── 数据存储（批量）
```

### 空间数据处理

- 空域数据：PostGIS 空间数据库
- 轨迹数据：时空数据库（支持时空查询）
- 三维可视化：Cesium.js + 3D Tiles

## 性能要求

| 指标 | 要求 |
|---|---|
| 并发飞行器 | ≥ 1000 |
| 遥测数据延迟 | < 1s |
| 冲突检测延迟 | < 100ms |
| 平台可用性 | ≥ 99.9% |
| 数据保留 | 飞行记录 ≥ 3 年 |

## 等保合规

| 级别 | 要求 |
|---|---|
| 等保 2.0 三级 | 安全管理制度、安全技术措施 |
| 密码合规 | 国密算法（SM2/SM3/SM4） |
| 数据合规 | 数据安全法、个人信息保护法 |
