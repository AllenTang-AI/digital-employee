# 代码审查标准 — code-review-standards

## 必检项（必须通过）

1. **禁止硬编码密钥** — API key、密码、token 必须使用环境变量或密钥管理
2. **输入校验** — 所有外部输入必须校验和清洗
3. **错误处理** — 禁止裸 `except:`，禁止吞掉异常不记录日志
4. **无调试代码** — 合并前移除 `print()`、`console.log()`、注释掉的代码
5. **测试覆盖** — 新功能必须有测试，bug 修复必须有回归测试

## 编码规范

### Python
- 遵循 PEP 8，最大行宽 120
- 所有公开函数带类型注解
- 所有公开类和函数带文档字符串
- 使用 `pathlib` 替代 `os.path`
- 优先用 `logging` 替代 `print`

### 通用
- 函数名：动词 + 名词，不超过 30 字符
- 单一职责：一个函数做 3 件事就拆分
- 禁止魔法数字：提取为命名常量
- 依赖：显式导入，禁止 `from x import *`

## 审查流程

1. **先自审** — 作者在请求审查前先看过自己的 diff
2. **关注上下文** — 审查者阅读相关代码，不只关注 diff
3. **NIT 与 BLOCK** — 建议前缀 `[NIT]`，必须改前缀 `[BLOCK]`
4. **解决对话** — 作者修改后逐个解决评论
5. **通过或要求修改** — 不要"通过但有意见" — 要么通过，要么要求修改
