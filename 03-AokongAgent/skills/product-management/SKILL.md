# 产品管理规范 — product-management

## PRD 模板结构

### 1. 产品概述
- 产品名称、版本、目标用户
- 核心价值主张（一句话说明白）
- 与竞品的差异化

### 2. 用户场景
- 按角色列出核心使用场景
- 每个场景：触发条件 → 用户操作 → 系统响应 → 结束状态

### 3. 功能需求
- 按模块分组，每个功能：
  - 功能描述
  - 用户故事（As a... I want... so that...）
  - 验收标准（Given/When/Then 格式）
  - 优先级（P0/P1/P2）

### 4. 非功能需求
- 性能指标
- 可靠性要求（MTBF）
- 环境规格（温度、抗风等）
- 法规合规

### 5. 依赖与风险
- 硬件依赖
- 第三方服务
- 已知风险

## 需求优先级 — Kano 模型

| 类型 | 特征 | 优先级 | 示例 |
|---|---|---|---|
| 基本型 | 没有就不行，有了不加分 | P0 | 飞行稳定性、电池安全 |
| 期望型 | 越多越好 | P1 | 续航时间、控制距离 |
| 兴奋型 | 有了很惊喜 | P2 | AI 避障、自动跟拍 |
| 无差异型 | 用户不在乎 | — | 包装颜色 |
| 反向型 | 做了反而不好 | ✗ | 过度复杂的操作界面 |

## 产品生命周期

1. **概念验证** — 技术可行性、市场需求验证
2. **MVP** — 最小可用产品，验证核心价值
3. **功能完善** — 补齐 P0/P1 需求
4. **量产优化** — 成本降低、可靠性提升
5. **迭代升级** — 用户反馈驱动的功能迭代
